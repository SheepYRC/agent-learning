1.1 什么是智能体？

核心要素：感知环境、自主决策、执行、反馈

以预设的模型作为经验，从简单到复杂，发展了 基于规则的反射、基于目标的规划、基于效用最大化等。能力源于人对世界和规则的显示总结和构建

后来出现了学习型智能体，通过机器学习得到决策策略。能力源于高质量特定数据
（AlphaGo Zero 是这一理念的一个里程碑式的成就。它在围棋这一复杂博弈中，通过强化学习发现了许多超越人类既有知识的有效策略。）

近几年，大语言模型驱动的新模式，通过海量数据，获得强大的涌现和泛化，更灵活更通用

不过这里也有一个问题，是复杂度的飙升，使得思考深度和运行成本、决策时间成正相关，追求反应性和规划性需要平衡


1.2 智能体的构成和运行原理

任务环境，通常以 PEAS 模型描述，且是具体 序贯和动态 的

Performance 性能度量
Environment 目标环境
Actuators   执行器
Sensors     传感器

核心机制：智能体循环
感知——思考（规划、工具）——行动——环境状态变化——新的感知


1.2.3 智能体的感知与行动

明确的交互协议，来规范信息交换。体现在每一次输出的结构化内容上，展示了推理和决策。
因此通常包含了两个部分；思考、行动

Thought: 用户想知道北京的天气。我需要调用天气查询工具。
Action: get_weather("北京")

> 这里的Action字段构成了对外部世界的指令。一个外部的解析器 (Parser) 会捕捉到这个指令，并调用相应的get_weather函数。

> 行动执行后，环境会返回一个结果。例如，get_weather函数可能返回一个包含详细天气数据的 JSON 对象。然而，原始的机器可读数据（如 JSON）通常包含 LLM 无需关注的冗余信息，且格式不符合其自然语言处理的习惯。

> 因此，感知系统的一个重要职责就是扮演传感器的角色：将这个原始输出处理并封装成一段简洁、清晰的自然语言文本，即观察。

Observation: 北京当前天气为晴，气温25摄氏度，微风。

> 这段Observation文本会被反馈给智能体，作为下一轮循环的主要输入信息，供其进行新一轮的Thought和Action。

综上所述，通过这个由 Thought、Action、Observation 构成的严谨循环，LLM 智能体得以将内部的语言推理能力，与外部环境的真实信息和工具操作能力有效地结合起来。
